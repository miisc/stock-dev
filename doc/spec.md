# 股票回测系统需求规格（Spec）

## 1. 文档目标

本文档定义股票回测系统的产品需求边界、业务规则、验收标准与里程碑交付物，面向A股日线回测场景，确保“结果可信、流程可复现、验收可执行”。

## 2. 产品目标与非目标

### 2.1 产品目标
1. 支持A股股票池的批量数据管理、批量回测与结果分析。
2. 为策略研究提供可复现的实验流程与标准化指标。
3. 提供面向研究人员的可筛选、可对比、可导出的回测结果视图。

### 2.2 非目标
1. 不提供实时交易与实盘下单能力。
2. 不覆盖多资产（港股、美股、期货、期权、外汇等）。
3. 不覆盖分钟级与Tick级高频场景。

## 3. 用户与典型任务

### 3.1 目标用户
1. 策略研究员：验证策略在A股历史样本中的表现。
2. 量化爱好者：快速比较不同股票池和参数组合。
3. 产品/分析人员：查看汇总指标并导出研究结论。

### 3.2 标准任务闭环
1. 创建或选择股票池。
2. 下载并校验历史数据。
3. 选择策略与参数并发起批量回测。
4. 查看汇总结果并筛选候选标的。
5. 保存实验记录，导出结果并用于后续复盘。

## 4. 业务范围

### 4.1 包含范围
1. 股票池管理：预置池与自定义池。
2. A股日线历史数据管理：批量下载、增量更新、质量校验。
3. 批量回测：面向股票池逐标的执行。
4. 结果分析：单标的指标、批量聚合、分布展示、导出。
5. 实验管理：保存回测任务元信息，实现可追溯与可复现。

### 4.2 排除范围
1. 实时行情接入与实时策略执行。
2. 多用户协作、权限体系、组织管理。
3. 分布式调度与集群部署。
4. 内置复杂机器学习训练流程。

## 5. 核心业务规则（必须统一口径）

### 5.1 交易规则口径
1. 市场范围：仅A股普通股票。
2. 交易制度：T+1。
3. 最小交易单位：按“手”交易，默认1手=100股。
4. 涨跌停处理：触及涨跌停时按不可成交处理（除非数据源明确可成交口径并在实验中声明）。
5. 停牌处理：停牌日不可成交，持仓按规则延续。
6. 除权除息：必须明确采用复权口径（前复权/后复权/不复权），并在结果中保留口径标签。
7. 上市/退市处理：仅在有效交易区间内参与回测，退市后不再交易。
8. ST处理：默认纳入；允许通过筛选条件排除。

### 5.2 成本规则口径
1. 成本项需可配置：佣金、印花税、过户费、最低佣金、滑点。
2. 买卖方向差异需可表达（如印花税仅卖出收取）。
3. 成本参数必须记录到实验快照，确保复现。

### 5.3 指标定义口径
1. 指标最小集合：总收益率、年化收益率、最大回撤、夏普比率、交易次数、胜率、盈亏比。
2. 指标所需参数（如无风险利率、年化因子）必须在系统中可见且可追溯。
3. 同一指标在单标的视图与汇总视图定义一致。

## 6. 功能需求（FR）

### FR-1 股票池管理
1. 系统应提供预置股票池：沪深300、上证50、创业板50、中证500、全A。
2. 系统应支持用户输入自定义股票代码列表。
3. 系统应支持保存并复用自定义股票池。
4. 验收标准：用户可在回测任务中直接复用任一历史股票池。

### FR-2 数据管理
1. 系统应支持按股票池批量下载日线数据。
2. 系统应支持时间范围设置与增量更新。
3. 系统应展示下载进度、失败数量与失败明细。
4. 系统应支持失败重试，并记录重试结果。
5. 验收标准：同一时间范围重复执行时，不重复拉取已存在有效数据。

### FR-3 数据质量校验
1. 系统应对缺失、重复、日期乱序、异常值进行校验并记录。
2. 系统应提供数据质量状态（通过/警告/失败）。
3. 系统应允许“仅对质量通过数据回测”或“带警告继续回测”的选择。
4. 验收标准：每次数据任务均可产出质量报告摘要。

### FR-4 策略与参数配置
1. 系统应提供策略选择与参数配置入口。
2. 系统应校验参数合法性（范围、类型、必填项）。
3. 系统应支持参数模板复用。
4. 验收标准：参数错误时任务不可启动，且给出可理解的提示。

### FR-5 批量回测任务
1. 系统应支持按股票池逐标的执行批量回测。
2. 系统应展示任务进度（已完成/总数/失败数）。
3. 系统应支持任务取消。
4. 系统应支持中断后续跑（至少从未完成标的继续）。
5. 验收标准：任务取消后，已完成结果可见，未完成标的状态明确。

### FR-6 结果分析与展示
1. 系统应提供批量汇总表：每只股票一行。
2. 汇总表应支持排序、筛选、字段选择。
3. 系统应提供Top N视图与整体分布视图。
4. 系统应提供单标的回测结果详情视图。
5. 验收标准：用户可在3步内从汇总定位到单标的详情。

### FR-7 导出与报告
1. 系统应支持导出批量汇总结果CSV。
2. 系统应支持导出单标的交易记录CSV。
3. 导出文件应包含字段说明、时间格式、数值精度约定。
4. 验收标准：导出文件可被常见表格工具直接打开并正确解析。

### FR-8 实验管理与可复现
1. 每次回测应形成一个“实验记录”。
2. 实验记录应包含：股票池快照、时间范围、策略与参数、成本参数、数据口径、运行时间。
3. 系统应支持按实验名称/时间检索历史记录。
4. 系统应支持至少两次实验结果对比。
5. 验收标准：同一实验快照重复执行，结果偏差在定义阈值内。

## 7. 非功能需求（NFR）

### NFR-1 性能
在以下基准前提下验收：
1. 基准数据规模：沪深300近5年日线。
2. 基准网络：稳定宽带网络，无额外代理限速。
3. 基准硬件：中端个人电脑。

性能目标：
1. 单股票5年数据下载：P95 ≤ 30秒。
2. 沪深300批量下载：P95 ≤ 30分钟。
3. 单股票5年回测：P95 ≤ 5秒。
4. 沪深300批量回测：P95 ≤ 30分钟。
5. 界面核心操作响应：P95 ≤ 2秒。

### NFR-2 稳定性
1. 长任务运行过程中不可因单标的失败导致整体崩溃。
2. 关键流程失败应可恢复或可重试。
3. 系统应提供可定位问题的错误信息与日志摘要。

### NFR-3 可用性
1. 新用户应可在30分钟内完成首次“下载-回测-导出”闭环。
2. 关键步骤应有状态提示与下一步引导。

### NFR-4 可追溯性
1. 所有回测结果应可追溯到实验快照。
2. 重要口径变更应在结果中可见（如复权口径、成本参数变化）。

## 8. 成功标准（可验收）

### 8.1 业务成功标准
1. 用户可完成完整研究闭环：建池、下载、回测、分析、导出、复盘。
2. 同口径重复回测结果一致性满足阈值要求。
3. 批量结果可支持快速筛选与对比，形成候选标的清单。

### 8.2 验收清单
1. 功能验收：FR-1 至 FR-8 全部通过。
2. 性能验收：NFR-1 指标满足P95目标。
3. 稳定性验收：异常场景（网络失败、数据异常、任务取消）全部可恢复或可解释。
4. 可追溯验收：任一结果可定位到完整实验快照。

## 9. 里程碑与交付物

### M1 数据管理里程碑
1. 交付范围：FR-1、FR-2、FR-3。
2. 验收产物：数据任务演示、数据质量报告样例、失败重试记录。

### M2 回测执行里程碑
1. 交付范围：FR-4、FR-5。
2. 验收产物：批量回测任务演示、任务取消与续跑演示、参数校验清单。

### M3 结果分析里程碑
1. 交付范围：FR-6、FR-7。
2. 验收产物：汇总看板演示、Top N与分布视图、CSV导出样例。

### M4 实验管理里程碑
1. 交付范围：FR-8、NFR-4。
2. 验收产物：实验快照样例、历史检索演示、两次实验对比报告。

## 10. 风险与假设

### 10.1 关键风险
1. 数据源变更或限流导致下载稳定性下降。
2. 交易规则口径不统一导致结果偏差。
3. 性能指标在低配设备下不达标。

### 10.2 前置假设
1. 数据源可持续提供A股日线历史数据。
2. 用户具备基础投资与指标理解能力。
3. 验收环境满足NFR基准前提。

## 11. 术语与口径说明

1. 批量回测：对股票池内标的按统一策略与参数执行回测。
2. 实验快照：保证结果复现所需的最小配置集合。
3. 质量通过数据：满足完整性与一致性校验的数据集合。
