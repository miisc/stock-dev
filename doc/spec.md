# 股票回测系统项目范围 (Spec)

## 项目概述

### 项目目标
开发一个基于Python的股票回测系统，专注于A股市场的策略回测和分析。该系统旨在为量化交易初学者和研究人员提供一个简单易用的回测平台，帮助他们验证交易策略的有效性。

### 项目背景
在量化交易领域，回测是验证策略有效性的关键步骤。

### 核心价值主张
1. **简单易用**: 提供直观的用户界面，降低使用门槛
2. **教育导向**: 代码结构清晰，便于学习和理解
3. **快速验证**: 支持快速策略回测，提高研究效率
4. **开源免费**: 完全开源，无使用限制

## 决策与权衡

### 技术选型决策

#### 数据源选择：AKShare
**决策**: 选择AKShare作为主要数据源
**理由**: 
- 免费开源，无API调用限制
- 数据覆盖面广，支持A股历史数据
- 社区活跃，维护及时
- Python原生支持，集成简单

**权衡**: 
- 优势：免费、易用、数据质量较好
- 劣势：相比付费数据源，数据更新可能稍有延迟
- 缓解措施：实现数据缓存机制，减少实时依赖

#### 数据库选择：SQLite
**决策**: 选择SQLite作为数据存储方案
**理由**:
- 轻量级，无需额外安装配置
- 适合中小规模数据存储
- Python内置支持，部署简单
- 文件型数据库，便于备份和迁移

**权衡**:
- 优势：零配置、便携性好、适合单机使用
- 劣势：并发写入性能有限，不适合大规模分布式应用
- 缓解措施：本项目主要面向单用户回测场景，SQLite完全满足需求

#### 用户界面选择：PyQt
**决策**: 选择PyQt作为桌面GUI框架
**理由**:
- 原生桌面应用，响应速度快
- 支持复杂交互（进度条、多标签页、表格排序等）
- 与Python生态完美集成
- 便于实现批量操作的异步进度反馈

**权衡**:
- 优势：性能好、交互能力强、适合批量操作场景
- 劣势：界面开发比Web框架稍复杂
- 缓解措施：使用Qt Designer辅助布局设计

### 功能范围决策

#### 策略复杂度：基础技术指标策略
**决策**: 专注于基础技术指标策略，不包含复杂机器学习策略
**理由**:
- 基础策略更容易理解和学习
- 计算简单，性能开销小
- 适合教学和演示场景
- 为后续扩展奠定基础

**权衡**:
- 优势：简单易懂、性能好、教育价值高
- 劣势：策略覆盖面有限，不适合高级用户
- 缓解措施：预留策略接口，支持用户自定义扩展

#### 数据频率：日线数据
**决策**: 仅支持日线数据回测，不支持分钟级或tick级数据
**理由**:
- 日线数据更容易获取和处理
- 适合中长期策略回测
- 降低系统复杂度和存储需求
- 符合初学者学习需求

**权衡**:
- 优势：数据获取简单、处理效率高、存储需求小
- 劣势：无法支持高频策略回测
- 缓解措施：明确项目定位，专注于中长期策略

## 项目范围

### 包含范围

#### 用户操作流程（App Operation Flow）

用户使用该系统的标准流程分为三步：

**第一步：数据管理（一次性操作）**
- 从预置股票池中选择目标范围（沪深300、上证50、创业板50、中证500、全部A股，或自定义代码列表）
- 设置历史数据的时间范围（开始/结束日期）
- 触发批量下载，系统显示实时下载进度（已下载 x/总数，失败列表）
- 数据保存至本地SQLite数据库；再次运行时只更新增量数据，不重复下载历史

**第二步：批量回测**
- 选择回测策略及参数（如双均线短周期=5、长周期=20）
- 选择回测股票池（复用数据管理中已下载的股票池，或自定义子集）
- 设置回测时间范围
- 触发批量回测，系统显示进度（已回测 x/总数）；支持取消

**第三步：结果汇总展示**
- 汇总统计表格：每只股票一行，列出总收益率、夏普比率、最大回撤、交易次数等指标
- 表格支持按任意列排序、筛选，快速定位最优/最劣标的
- Top N 最佳股票图表展示
- 策略整体胜率与平均收益统计
- 支持导出全部结果到 CSV

#### 核心功能模块
1. **股票池管理模块**
   - 维护预置股票池：沪深300、上证50、创业板50、中证500、全部A股
   - 通过AKShare获取各指数最新成分股列表
   - 支持自定义股票代码列表

2. **数据获取模块**
   - 批量从AKShare获取A股日线数据（按股票池）
   - 数据清洗和预处理
   - 本地SQLite存储；支持增量更新
   - 批量下载进度反馈、失败重试、失败列表记录

2. **策略框架模块**
   - 策略基类定义
   - 双均线策略实现
   - 布林带策略实现
   - RSI策略实现
   - 策略参数配置

3. **回测引擎模块**
   - 批量回测调度：对股票池内每只股票依次执行回测
   - 前向遍历回测逻辑
   - 交易执行模拟
   - 交易成本计算
   - 仓位管理

4. **评估分析模块**
   - 单股票绩效指标计算
   - **批量回测结果聚合**：汇总表格、排序、筛选
   - 结果可视化（单股票图表 + 整体分布图）
   - 交易记录导出（单股票及全量 CSV）

5. **用户界面模块**
   - 股票池选择界面（预置池 + 自定义）
   - 批量数据下载界面（含进度条）
   - 策略配置与批量回测控制界面（含进度条）
   - 批量结果汇总展示界面（可排序表格 + 图表）

#### 支持功能
- 错误处理和日志记录
- 配置文件管理
- 单元测试覆盖
- 基础文档

### 明确排除范围

#### 功能排除
1. **多资产类别支持**
   - 仅支持A股，不支持港股、美股、期货、外汇等
   - 理由：简化系统复杂度，专注单一市场

2. **高频交易支持**
   - 仅支持日线数据，不支持分钟级或tick级数据
   - 理由：降低系统复杂度和存储需求

3. **实时交易功能**
   - 仅支持历史回测，不支持实时交易
   - 理由：避免监管风险，专注回测功能

4. **复杂衍生品支持**
   - 不支持期权、期货等衍生品回测
   - 理由：简化系统复杂度

5. **高级机器学习策略**
   - 不内置复杂机器学习策略
   - 理由：保持系统简单性，专注基础策略

#### 技术排除
1. **分布式架构**
   - 采用单机架构，不支持分布式部署
   - 理由：简化部署和维护

2. **实时数据流**
   - 不支持实时数据推送和处理
   - 理由：专注历史回测场景

3. **复杂用户管理**
   - 不支持多用户、权限管理等复杂功能
   - 理由：简化系统设计

## 风险与缓解措施

### 主要风险
1. **数据源稳定性风险**
   - 风险：AKShare API可能不稳定或变更
   - 影响：数据获取失败
   - 缓解：实现重试机制，准备备用数据源

2. **性能风险**
   - 风险：大量数据回测可能超时
   - 影响：用户体验差
   - 缓解：数据库优化，算法优化

3. **需求变更风险**
   - 风险：开发过程中需求可能变更
   - 影响：进度延期
   - 缓解：明确范围，变更控制流程

### 应急计划
1. **数据源问题**
   - 短期：使用本地缓存数据
   - 长期：集成其他免费数据源

2. **性能问题**
   - 短期：优化算法，减少数据量
   - 长期：考虑升级数据库或架构

3. **进度延期**
   - 短期：调整功能优先级
   - 长期：增加资源或延长周期
